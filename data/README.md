# Data Module - 数据采集与预处理模块

│ │ 1. packet_capture.py (数据包捕获) │ │

## 二、目录功能介绍

| 功能模块       | 职责                      | 关键特性                                 |
| -------------- | ------------------------- | ---------------------------------------- |
| **数据包捕获** | 从网卡实时抓取原始网络包  | BPF过滤器、异步回调、批量获取            |
| **流聚合**     | 将数据包按五元组聚合成流  | 双向流量关联、TCP时序分析、流超时管理    |
| **特征提取**   | 从流中提取24个模型特征    | 全局统计维护、特征验证                   |
| **数据预处理** | 编码+标准化，生成模型输入 | LabelEncoder、StandardScaler、缺失值处理 |

## 三、各文件详细说明

### 1. packet_capture.py - 数据包捕获器

**作用：** 实时捕获网络数据包，从指定网卡抓取原始网络流量，提取每个数据包的基本信息。

**输出数据结构：**

| 字段              | 类型   | 说明                    | 示例值                 |
| ----------------- | ------ | ----------------------- | ---------------------- |
| `timestamp`       | float  | 捕获时间戳              | 1703123456.789         |
| `src_ip`          | str    | 源IP地址                | "192.168.1.100"        |
| `dst_ip`          | str    | 目标IP地址              | "8.8.8.8"              |
| `src_port`        | int    | 源端口                  | 54321                  |
| `dst_port`        | int    | 目标端口                | 80                     |
| `protocol`        | str    | 协议类型                | "tcp" / "udp" / "icmp" |
| `payload`         | bytes  | 数据负载                | b'GET / HTTP/1.1...'   |
| `payload_preview` | str    | payload预览（十六进制） | "474554202f204854..."  |
| `ttl`             | int    | 生存时间                | 64                     |
| `length`          | int    | 包长度（字节）          | 1500                   |
| `raw_packet`      | Packet | 原始scapy包对象         | -                      |

### 2. flow_aggregator.py - 流聚合器

**作用：** 将零散的数据包按五元组聚合成完整的网络流，计算流级别的统计特征。

**五元组标识结构：**

| 字段       | 类型 | 说明       | 示例值          |
| ---------- | ---- | ---------- | --------------- |
| `src_ip`   | str  | 源IP地址   | "192.168.1.100" |
| `dst_ip`   | str  | 目标IP地址 | "8.8.8.8"       |
| `src_port` | int  | 源端口     | 54321           |
| `dst_port` | int  | 目标端口   | 80              |
| `protocol` | str  | 协议类型   | "tcp"           |

**输出数据结构（FlowStats）：**

| 分类         | 字段               | 类型  | 说明                       | 对应模型特征     |
| ------------ | ------------------ | ----- | -------------------------- | ---------------- |
| **包计数**   | `forward_packets`  | int   | 源->目的包数               | spkts            |
|              | `backward_packets` | int   | 目的->源包数               | dpkts            |
| **字节计数** | `forward_bytes`    | int   | 源->目的字节数             | sbytes           |
|              | `backward_bytes`   | int   | 目的->源字节数             | dbytes           |
| **TTL统计**  | `forward_ttl`      | int   | 平均源TTL                  | sttl             |
|              | `backward_ttl`     | int   | 平均目的TTL                | dttl             |
| **丢包统计** | `forward_loss`     | int   | 源丢包数                   | sloss            |
|              | `backward_loss`    | int   | 目的丢包数                 | dloss            |
| **抖动统计** | `forward_jitter`   | float | 源抖动（时间间隔标准差）   | sjit             |
|              | `backward_jitter`  | float | 目的抖动                   | djit             |
| **TCP时序**  | `tcp_rtt`          | float | 往返时间（秒）             | tcprtt           |
|              | `tcp_synack`       | float | SYN-ACK时间（秒）          | synack           |
|              | `tcp_ackdat`       | float | ACK数据时间（秒）          | ackdat           |
| **应用层**   | `trans_depth`      | int   | 传输深度（HTTP事务深度）   | trans_depth      |
|              | `is_ftp_login`     | bool  | 是否为FTP登录              | is_ftp_login     |
|              | `ct_flw_http_mthd` | int   | HTTP方法计数               | ct_flw_http_mthd |
|              | `is_sm_ips_ports`  | bool  | 是否为小IP/端口            | is_sm_ips_ports  |
| **元数据**   | `start_time`       | float | 流开始时间                 | -                |
|              | `last_time`        | float | 最后包时间                 | -                |
|              | `duration`         | float | 流持续时间（派生属性）     | -                |
|              | `state`            | str   | 流状态（CON/FIN/INT/no等） | state            |

### 3. feature_extractor.py - 特征提取器

**作用：** 从流统计中提取模型所需的24个特征，并维护需要全局上下文的统计特征。

**输出数据结构（24个特征字典）：**

| 序号 | 特征键名           | 类型  | 说明                     | 取值范围/示例                         |
| ---- | ------------------ | ----- | ------------------------ | ------------------------------------- |
| 1    | `proto`            | str   | 协议类型                 | "tcp" / "udp" / "icmp" / "ip"         |
| 2    | `state`            | str   | 流状态                   | "CON" / "FIN" / "INT" / "no"          |
| 3    | `sbytes`           | int   | 源->目的字节数           | 0 - 65535+                            |
| 4    | `dbytes`           | int   | 目的->源字节数           | 0 - 65535+                            |
| 5    | `sttl`             | int   | 源TTL                    | 1 - 255                               |
| 6    | `dttl`             | int   | 目的TTL                  | 1 - 255                               |
| 7    | `sloss`            | int   | 源丢包数                 | 0 - 65535                             |
| 8    | `dloss`            | int   | 目的丢包数               | 0 - 65535                             |
| 9    | `spkts`            | int   | 源包数                   | 0 - 65535                             |
| 10   | `dpkts`            | int   | 目的包数                 | 0 - 65535                             |
| 11   | `sjit`             | float | 源抖动（秒）             | 0.0 - 1.0                             |
| 12   | `djit`             | float | 目的抖动（秒）           | 0.0 - 1.0                             |
| 13   | `tcprtt`           | float | TCP往返时间（秒）        | 0.0 - 60.0                            |

| 23   | `ct_flw_http_mthd` | int   | HTTP方法计数             | 0 - 65535                             |

## 四、各文件输出总结

| 文件                   | 输出类型         | 输出形状           | 输出格式             | 后续用途                          |
| ---------------------- | ---------------- | ------------------ | -------------------- | --------------------------------- |
| `packet_capture.py`    | `CapturedPacket` | 单个包             | Python对象（数据类） | 输入到流聚合器进行五元组分组合并  |
| `flow_aggregator.py`   | `FlowStats`      | 单个流             | Python对象（数据类） | 输入到特征提取器计算24个特征      |
| `feature_extractor.py` | `Dict[str, any]` | 24个键值对         | Python字典           | 输入到预处理器进行编码和标准化    |
| `preprocessor.py`      | `np.ndarray`     | (1, 24) 或 (n, 24) | NumPy浮点数组        | 输入到XGBoost模型进行威胁概率预测 |

## 五、后续调用思路

### 整体集成思路

1. **初始化阶段**
   - 创建 PacketCapturer 实例，配置网卡和BPF过滤器
   - 创建 FlowAggregator 实例，设置流超时时间
   - 创建 FeatureExtractor 实例
   - 创建 DataPreprocessor 实例，指定模型文件目录

2. **数据采集循环**
   - 启动 PacketCapturer 开始捕获数据包
   - 通过回调函数或队列获取 CapturedPacket 对象
   - 将每个数据包传入 FlowAggregator 进行聚合

3. **流完成处理**
   - FlowAggregator 返回完成的 FlowStats 对象（流超时或收到FIN/RST）
   - 将 FlowStats 传入 FeatureExtractor 提取24个特征字典
   - 将特征字典传入 DataPreprocessor 进行编码和标准化
   - 得到 NumPy 数组后输入 XGBoost 模型进行预测

4. **资源释放**
   - 停止 PacketCapturer
   - 调用 FlowAggregator.flush_all() 处理剩余流

### 与检测引擎的协作思路

- 该模块输出 NumPy 数组后，由检测引擎调用 XGBoost 模型获取威胁概率
- 检测引擎根据阈值判断：威胁（>=0.5）、模糊（0.3-0.5）、正常（<0.3）
- 模糊行为需上报前端由管理员判断
- 管理员确认的威胁需通过规则仓库模块添加到 MySQL 规则库

### 性能优化思路

- 高流量场景下使用批量获取（get_packet_batch）减少函数调用开销
- 使用 FlowAggregator.flush_all() 定期清理超时流，避免内存积压
- 特征提取和预处理可放入独立线程池，避免阻塞数据包捕获

### 错误处理思路

- 模型文件不存在时降级为仅规则匹配模式
- 编码器遇到未知类别时使用-1填充，避免程序崩溃
- 数据包捕获异常时记录日志并尝试重新连接网卡
